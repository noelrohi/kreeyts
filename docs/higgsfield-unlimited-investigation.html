Investigation note · 2026-07-01

Higgsfield “Unlimited Generate” in Assetwell

Summary of GitHub issue #3: whether Assetwell can expose Higgsfield's web-only Unlimited mode for generation without spending credits.

Decision

Do not add an Assetwell Unlimited toggle while Assetwell uses the official @higgsfield/cli. The current CLI does not expose a supported unlimited, relaxed, standard-queue, or subscription-bucket flag.

A private web-app mechanism appears to exist, but Higgsfield's public documentation says Unlimited is available on higgsfield.ai only and not through CLI, MCP, Canvas, or Supercomputer. Treat private web routes as unsupported and fragile.

No CLI flag Web-only entitlement Private API not product-safe Track upstream

What was verified

  • Vendored package: @higgsfield/cli@1.0.2.
  • CLI rejects obvious params: --unlimited, --use_unlim, --queue standard, and --use_free_gens.
  • hf model get nano_banana_2 exposes model params only, not billing mode or queue selection.
  • generate cost still returns credit costs for eligible-looking requests.

What exists on the web app

The Higgsfield web bundle contains separate unlimited job-set identifiers and UI copy for an “Unlimited mode” toggle. Examples observed in public web assets include:

  • nano_banana_pro_unlimited
  • gpt_image_2_unlimited
  • kling_3_unlimited
  • seedance_unlimited / seedance_2_unlimited

Surface comparison

Surface Unlimited support Assetwell implication
Higgsfield web app Yes, for entitled models/plans via the web UI. Works for humans in browser; not directly reusable by Assetwell's CLI wrapper.
Official Higgsfield CLI No public flag or model param in 1.0.2. Assetwell cannot safely pass an unlimited option today.
Official MCP / agent docs Documented as credit-based. Same limitation as CLI for product expectations.
Private web backend Appears to include use_unlim and unlimited job-set routes. Do not ship against it without explicit Higgsfield support and auth guidance.

CLI probes

hf generate cost nano_banana_2 --prompt test --unlimited true
# Error: Unknown params: unlimited

hf generate cost nano_banana_2 --prompt test --use_unlim true
# Error: Unknown params: use_unlim

hf generate cost nano_banana_2 --prompt test --queue standard
# Error: Unknown params: queue

Cost probes

hf --json generate cost seedance_2_0 \
  --prompt test --mode fast --resolution 720p
# { "credits": 17.5 }

hf --json generate cost nano_banana_2 \
  --prompt test --resolution 2k
# { "credits": 2 }

Public docs signal

Higgsfield's Seedance Unlimited post states Unlimited is available only on higgsfield.ai, and that MCP, CLI, Canvas, and Supercomputer generations consume credits regardless of Unlimited status.

Recommended product behavior

  1. Keep issue #3 open as blocked upstream until Higgsfield exposes a supported CLI/API flag or entitlement-aware cost endpoint.
  2. Do not expose a fake toggle in Assetwell. It would imply savings that the CLI cannot guarantee.
  3. If users ask, explain: “Assetwell uses Higgsfield's CLI. Higgsfield currently keeps Unlimited mode on the web app only, so Assetwell generations use credits.”
  4. Revisit when upstream issues requesting --unlimited or subscription-aware costs are resolved.
